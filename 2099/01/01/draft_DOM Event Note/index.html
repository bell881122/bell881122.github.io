【DOM】關於 Event 的 Bubbling/Capturing/Delegation

發表於2099-01-01|更新於2021-06-06
|閱讀量:

Event Bubbling（事件冒泡）

Event Capturing（事件捕獲）

觸發事件的順序是
先 Capturing（由外向內）朝內捕獲
來到 target 本身
再 Bubbling （由內向外）傳遞出去

加上 event.stopPropagation() 即可防止事件傳遞
中斷上述過程

Event Delegation（事件委派）

當多個節點呼叫相同的 event handler 時
利用 Event Bubbling 的特性
直接在祖先節點上統一執行 event handler
就不需要於後代節點上分別處理了
